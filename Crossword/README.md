# Crossword Puzzle Solver

A constraint-satisfaction crossword puzzle solver in Perl with optional
LLM-assisted hint generation. Solves puzzles from ASCII grid files using a
local dictionary — no internet access required for the core solver.

## Quick Start

```bash
# Solve a puzzle from an ASCII grid file
perl solve.pl examples/medium_5x5.txt

# Convert a PDF crossword to ASCII and solve it
python3 pdf_to_grid.py "WSJ Crosswords/XWD01032022.pdf" -o .
perl solve.pl XWD01032022.txt

# Full pipeline: PDF → hints → solve (requires ollama)
perl solve_with_hints.pl "WSJ Crosswords/XWD01032022.pdf" -v
```

## Components

| Script | Language | Purpose |
|--------|----------|---------|
| `solve.pl` | Perl | Core crossword solver (AC-3 + backtracking) |
| `pdf_to_grid.py` | Python | Convert PDF crosswords to ASCII grid format |
| `get_hints.py` | Python | Extract clues from PDF and query LLM for candidates |
| `solve_with_hints.pl` | Perl | Full pipeline wrapper (PDF → grid → hints → solve) |
| `test_all.pl` | Perl | Batch test harness for solving many puzzles |

## Solver (`solve.pl`)

```
perl solve.pl [options] <puzzle_file>
```

### Options

| Flag | Description |
|------|-------------|
| `-d`, `--dict FILE` | Path to dictionary file (default: `dictionary.txt` in script dir) |
| `-H`, `--hints FILE` | Path to hints file with LLM-generated candidates |
| `-m`, `--max N` | Maximum number of solutions to find (default: 1) |
| `-v`, `--verbose` | Show slot analysis, candidate counts, and timing |
| `-h`, `--help` | Show usage help |

### Examples

```bash
# Solve with verbose output
perl solve.pl -v examples/standard_7x7.txt

# Find up to 3 solutions
perl solve.pl -m 3 examples/open_5x5.txt

# Solve with LLM-generated hints (much faster for hard puzzles)
perl solve.pl --hints=puzzle.hints puzzle.txt
```

### Two-Phase Solving (with hints)

When a hints file is provided, the solver uses a two-phase strategy:

1. **Phase 1 (hints-only)**: Restricts hinted slots to *only* LLM candidates.
   If the LLM was mostly correct, this solves nearly instantly.
2. **Phase 2 (full dictionary)**: Falls back to the full dictionary but
   prioritizes hint words in the search order, dramatically reducing
   backtracking time.

This two-phase approach converts many previously-timed-out puzzles into
sub-10-second solves.

### Algorithm

1. **Parse** the grid and extract word slots (runs of 2+ non-black cells)
2. **Build domains** from dictionary words matching each slot's length and clue letters
3. **Arc consistency (AC-3)** prunes candidates that can't satisfy crossing constraints
4. **Backtracking search** with MRV heuristic (smallest domain first) and forward checking
5. When hints are provided, hint words are tried before dictionary words

## Input Format

Plain text file where each line is a row of the grid:

| Character | Meaning |
|-----------|---------|
| `#` | Black square |
| `_` | Blank cell (to be solved) |
| `A`-`Z` or `a`-`z` | Pre-filled letter clue |

All rows must be the same length. Example:

```
#_#_#
STA_E
#_#_#
OP_NS
#_#_#
```

## Hints File Format

Generated by `get_hints.py` or created manually. Each non-comment line
specifies a direction, clue number, and one or more candidate words:

```
# Lines starting with # are comments
A 1 ASH OAK BAT       # Across 1: three candidates
D 5 GATOR CROC        # Down 5: two candidates
A 14 HELLO WORLD      # Across 14
```

Candidates are listed in priority order (best guess first). Words that
don't match the slot length are automatically filtered out.

## PDF Grid Converter (`pdf_to_grid.py`)

Extracts grids from WSJ-style crossword PDFs (vector-based, with filled
rectangles for black squares).

**Requirements**: Python 3 + PyMuPDF (`pip install pymupdf`)

```bash
# Convert a single PDF (prints to stdout)
python3 pdf_to_grid.py "WSJ Crosswords/XWD01032022.pdf"

# Convert to a specific output directory
python3 pdf_to_grid.py "WSJ Crosswords/XWD01032022.pdf" -o converted/

# Batch convert all PDFs in a directory
python3 pdf_to_grid.py --batch "WSJ Crosswords/" -o "WSJ Crosswords/"

# Verbose mode shows grid dimensions and detection details
python3 pdf_to_grid.py -v "WSJ Crosswords/XWD01032022.pdf"
```

Barred/variety puzzles (e.g., Saturday puzzles) that lack standard black
squares are automatically detected and skipped.

## LLM Hint Generator (`get_hints.py`)

Extracts crossword clues from PDF text and queries a local ollama instance
to generate candidate answers.

**Requirements**: Python 3 + PyMuPDF, ollama running locally

```bash
# Generate hints for a single puzzle
python3 get_hints.py "WSJ Crosswords/XWD01032022.pdf" -o puzzle.hints

# Use a specific model with more candidates
python3 get_hints.py puzzle.pdf -o puzzle.hints -m llama3.1:8b -n 5

# Batch mode for a whole directory
python3 get_hints.py --batch "WSJ Crosswords/" -o "WSJ Crosswords/"

# Verbose shows clue matching and LLM responses
python3 get_hints.py puzzle.pdf -o puzzle.hints -v
```

### Options

| Flag | Description |
|------|-------------|
| `-o FILE` | Output hints file path |
| `-m MODEL` | Ollama model name (default: `qwen2.5:7b`) |
| `-n N` | Number of candidates per clue (default: 3) |
| `--grid FILE` | Pre-converted grid file (skips PDF grid extraction) |
| `--batch DIR` | Process all PDFs in directory |
| `-v` | Verbose output |

### Supported Models

Any ollama model works. Tested with:

- `qwen2.5:7b` — fast, good for factual/trivia clues (recommended)
- `llama3.1:8b` — slightly slower, competitive quality
- `deepseek-r1:7b` — reasoning model, slower but sometimes better for wordplay

## Full Pipeline (`solve_with_hints.pl`)

Runs the complete pipeline from PDF to solved grid in one command:

```bash
perl solve_with_hints.pl [options] <puzzle.pdf>
```

### Options

| Flag | Description |
|------|-------------|
| `-n`, `--candidates N` | Candidates per clue from LLM (default: 3) |
| `-m`, `--model MODEL` | Ollama model (default: `qwen2.5:7b`) |
| `-t`, `--timeout SECS` | Solver timeout (default: 60) |
| `--skip-hints` | Use existing `.hints` file if present |
| `-v`, `--verbose` | Show progress |
| `-h`, `--help` | Show help |

### Example

```bash
# Solve a PDF crossword end-to-end
perl solve_with_hints.pl "WSJ Crosswords/XWD01042022.pdf" -v

# Reuse previously generated hints
perl solve_with_hints.pl "WSJ Crosswords/XWD01042022.pdf" --skip-hints
```

## Test Harness (`test_all.pl`)

Batch-solves all `.txt` grid files in a directory and reports statistics:

```bash
# Solve all converted puzzles (30s timeout per puzzle)
perl test_all.pl "WSJ Crosswords/"

# Custom timeout
perl test_all.pl --timeout=60 "WSJ Crosswords/"
```

Output: writes `*_solved.txt` files for successful solves and prints a
summary with solve rate, timeouts, and timing statistics.

## Dictionary

The included `dictionary.txt` contains ~370K English words from
[dwyl/english-words](https://github.com/dwyl/english-words). You can
substitute any newline-delimited word list with the `-d` flag.

## Example Puzzles

| File | Description |
|------|-------------|
| `examples/mini_5x5.txt` | 5×5, heavily clued, instant solve |
| `examples/mini_clued.txt` | 5×5 checkerboard with letter clues |
| `examples/medium_5x5.txt` | 5×5 alternating pattern, moderate clues |
| `examples/open_5x5.txt` | 5×5 completely open (no clues) |
| `examples/challenge_5x5.txt` | 5×5 with a single clue letter |
| `examples/standard_7x7.txt` | 7×7 standard crossword pattern, no clues |

## Performance

Tested on 243 WSJ crossword puzzles (15×15 standard grids):

| Mode | Solve Rate | Avg Time |
|------|-----------|----------|
| Dictionary only | 68.7% (167/243) | 13.3s |
| With LLM hints | Significantly improved | — |

Puzzles that timeout without hints often solve in under 10 seconds when
LLM candidates are available to guide the search.
